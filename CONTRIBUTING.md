# 为 RD-Agent 做贡献

我们欢迎对改进 RD-Agent 的贡献和建议。无论是解决问题、修复错误、增强文档，甚至是纠正拼写错误，每一个贡献都是有价值的，并有助于改进项目。

## 入门

要开始，您可以浏览问题列表或通过运行以下命令在代码库中搜索 `TODO:` 注释：
```sh
grep -r "TODO:"
```

## 如何贡献

1. **Fork 仓库**：在 GitHub 上创建仓库的一个 fork。
2. **克隆仓库**：将您 fork 的仓库克隆到本地计算机。
   ```sh
   git clone https://github.com/your-username/RD-Agent.git
   ```
3. **创建分支**：为您的更改创建一个新分支。
   ```sh
   git checkout -b feature/your-feature-name
   ```
4. **进行更改**：对代码库进行更改。
5. **提交更改**：使用描述性的提交消息提交您的更改。
   ```sh
   git commit -m "您的更改说明"
   ```
6. **推送更改**：将您的更改推送到您 fork 的仓库。
   ```sh
   git push origin feature/your-feature-name
   ```
7. **确保 CI 通过**：确保您的代码通过 GitHub 上的自动 CI 检查。
8. **创建拉取请求**：从您 fork 的仓库向主仓库创建一个拉取请求。

## 行为准则

在您与项目的所有互动中，请遵守[行为准则](CODE_OF_CONDUCT.md)。

## 报告问题

如果您遇到任何问题或有改进建议，请在 GitHub 上提出问题。

## 指南

- 确保您的代码遵循项目的编码标准。
- 编写清晰简洁的提交消息。
- 根据需要更新文档。
- 在提交拉取请求之前，请彻底测试您的更改。

感谢您为 RD-Agent 做出贡献！
